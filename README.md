## Оптимизация планирования туров в городе

### Описание задачи
Необходимо спланировать минимальное количество дневных прогулок для посещения всех достопримечательностей города за ограничение по расстоянию (максимум 20 км в день при скорости 5 км/ч).

### Алгоритм решения

#### 1. **Dijkstra's Algorithm** (Поиск кратчайших путей)
- Вычисляем кратчайшие пути между всеми парами достопримечательностей
- Временная сложность: O(k · (m + n) · log n), где k - количество достопримечательностей

#### 2. **Dynamic Programming with Bitmasks** (Уравнение Беллмана)
- Состояние: `dp[mask]` = минимальное число дней для посещения достопримечательностей, соответствующих битам в mask
- Переход: для каждого непосещённого подмножества, достижимого за один день (≤ 20км)
- Уравнение Беллмана: 
  ```
  dp[new_mask] = min(dp[new_mask], dp[mask] + 1)
  ```
- Временная сложность: O(3^k), так как перебираются все подмножества

#### 3. **Greedy Path Construction** (Жадное построение маршрутов)
- На каждый день выбираем достопримечательности, сортируя их по расстоянию от начальной точки
- Восстанавливаем полный путь между точками через Dijkstra

### Ключевые оптимизации

1. **Битовые маски** - эффективное представление подмножеств достопримечательностей
2. **Предварительный расчёт расстояний** - избегаем повторных вычислений Dijkstra
3. **Перебор подмножеств** - используем стандартный трюк `(submask - 1) & reachable`
4. **Восстановление пути** - только когда нужно вывести результат

### Сложность

- **Время**: O(3^k + k·(m+n)·log n)
- **Память**: O(k² + n + m)

Где:
- k - количество достопримечательностей
- n - количество перекрестков
- m - количество улиц

### Максимальные оценки

Оптимальное решение дает максимум баллов: `max(0, 100 - d)`, где d - число дней.
